Getting Started

Get SwiftRE running and execute your first ruleset in three steps.

What is SwiftRE?

SwiftRE is a lightweight business rules engine. You write rules in an English-like language, submit data as JSON, and get back structured events — errors, warnings, information messages, and custom signals. It's designed for form validation with immediate feedback, but works for any scenario where you need to externalize business logic from application code.

Prerequisites

You need one of the following:

  • Docker — the simplest way to get started
  • Swift 6.0+ toolchain — if you prefer to build from source

Quick Start

  1. Start the server

    With Docker Compose:

    docker compose up -d

    Or build from source:

    swift run SwiftREServer serve

    The API is now available at http://localhost:8080.

  2. Create a ruleset 
    curl -X POST http://localhost:8080/api/v1/rulesets \
  -H "Content-Type: application/json" \
  -d '{
    "id": "contact-form",
    "ruleText": "RULE \"Email Required\"\nWHEN email IS NOT SET\nTHEN ERROR ON email \"Email is required\"\n\nRULE \"Age Check\" PRIORITY 10\nWHEN age IS SET AND age IS LESS THAN 18\nTHEN ERROR ON age \"Must be 18 or older\""
  }'

    Response:

    {
  "id": "contact-form",
  "ruleCount": 2,
  "ruleNames": ["Age Check", "Email Required"],
  "hasSchema": false,
  "createdAt": "2025-01-15T10:30:00Z",
  "updatedAt": "2025-01-15T10:30:00Z"
}
  3. Execute the ruleset 
    curl -X POST http://localhost:8080/api/v1/rulesets/contact-form/execute \
  -H "Content-Type: application/json" \
  -d '{
    "facts": {
      "firstName": "Jane",
      "age": 16,
      "email": null
    }
  }'

    Response:

    {
  "events": [
    {
      "kind": "error",
      "ruleName": "Age Check",
      "field": "age",
      "message": "Must be 18 or older"
    },
    {
      "kind": "error",
      "ruleName": "Email Required",
      "field": "email",
      "message": "Email is required"
    }
  ],
  "facts": {
    "firstName": "Jane",
    "age": 16,
    "email": null
  },
  "iterations": 1,
  "terminationReason": "complete",
  "summary": {
    "errors": 2,
    "warnings": 0,
    "infos": 0,
    "modifications": 0,
    "total": 2
  }
}

Understanding the Response

The execution response contains:

FieldDescription
eventsArray of events produced by fired rules. Each has a kind, ruleName, and message. Field-targeted events include a field property.
factsThe complete working memory state after execution. If any SET actions fired, modified values appear here.
iterationsHow many evaluation passes were performed (1 for single-pass mode).
terminationReasonWhy execution stopped: complete, maxIterationsReached, or cycleDetected.
summaryEvent counts by kind, plus a total.
