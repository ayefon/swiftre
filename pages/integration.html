Form Integration

swiftre-validate.js connects HTML forms to the SwiftRE API for live, progressive validation with field highlighting.

Overview

The validation script provides a bridge between standard HTML forms and the SwiftRE execution API. When a user interacts with form fields, the script:

  1. Tracks which fields the user has touched ("dirty" fields)
  2. Debounces rapid changes (500ms delay) to avoid excessive API calls
  3. Collects all form values as JSON facts
  4. Executes the selected ruleset via the API
  5. Displays only events relevant to dirty fields
  6. Highlights specific fields with error/warning/info styling

Required HTML Elements

The script expects these elements on the page:

ElementPurpose
<form id="form">The form to validate. Field name attributes become fact paths.
<select id="ruleset-selector">Dropdown populated with available rulesets from the API.
<div id="messages">Container where validation messages are rendered.
<div id="status">Status indicator (shows API connection state).

Adding to Your Page

<!-- Configuration -->
<script>
  var swiftreUrl = "http://localhost:8080/api/v1";
  var swiftreRulesetId = null; // auto-select from dropdown
</script>

<!-- Validation panel -->
<div id="status"></div>
<select id="ruleset-selector"></select>
<div id="messages"></div>
<button type="button" onclick="SwiftREValidation.validateAll()">
  Validate All
</button>

<!-- The script -->
<script src="swiftre-validate.js"></script>

Fact Collection

The script iterates over all <input>, <select>, and <textarea> elements in the form. Each element's name attribute becomes a fact path, and the value is coerced:

Input TypeCoercion
Empty stringnull
Checkboxtrue / false
Numeric stringNumber (integer or float)
OtherString as-is

Progressive Validation

The script uses a progressive validation approach to avoid overwhelming the user with messages for untouched fields:

Dirty-Field Tracking

A field becomes "dirty" when the user interacts with it (changes its value). Only events relevant to dirty fields are displayed. This means a form with 10 empty required fields won't show 10 error messages until the user has actually visited those fields.

Debounce

After each field change, validation is delayed by 500ms. If the user changes another field within that window, the timer resets. This prevents rapid-fire API calls while typing or tabbing through fields.

Validate All

The "Validate All" button (calling SwiftREValidation.validateAll()) marks every field as dirty and runs validation immediately. This is useful as a final check before form submission.

Field Highlighting

When events are rendered, the script attempts to highlight the corresponding form field:

  1. Explicit targeting: If the event has a field property (from ERROR ON field syntax), that field is highlighted directly.
  2. Message heuristic: If no explicit field is set, the script checks if the event message mentions any dirty field's name or value.

Best practice: Use the ON field syntax in your rules for reliable field highlighting. The message heuristic is a fallback for simpler rules where the field name appears in the message text.

Event Filtering

Only error, warning, and info events are displayed. Other event kinds (dataModification, signal, etc.) are filtered out of the UI but remain in the API response for programmatic use.
