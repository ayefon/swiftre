Rule Language Reference

SwiftRE rules are written in an English-like language. Each rule has a name, an optional priority, a condition (WHEN), and one or more actions (THEN).

Rule Structure

RULE "Rule Name" [PRIORITY N]
WHEN <condition>
THEN <action> [AND <action>]*
PartRequiredDescription
RULE "name"YesUnique name in double quotes. Used in event output to identify which rule fired.
PRIORITY NNoInteger priority. Higher values evaluate first. Default is 0.
WHENYesThe condition that must be true for the rule to fire.
THENYesOne or more actions, separated by AND.

Priority and Evaluation Order

Rules are evaluated in descending priority order. Rules with the same priority are evaluated in the order they appear in the ruleset. In iterative mode, priority determines which rules re-fire first when facts change.

RULE "Critical Check" PRIORITY 100
WHEN status IS "blocked"
THEN ERROR "Processing is blocked"

RULE "Standard Check"
WHEN status IS "pending"
THEN INFO "Awaiting review"

Field References

Fields refer to values in the working memory (the JSON facts you submit). There are two ways to reference a field:

Bare identifiers

Single-word field names can be used directly:

WHEN age IS GREATER THAN 18
WHEN status IS "active"

Dotted paths

Nested JSON objects are flattened to dot-notation. Use backticks for paths with dots or spaces:

WHEN `address.city` IS "Springfield"
WHEN `customer.email` IS NOT SET

Tip: Backtick-quoted fields can contain any characters, including spaces and special characters. Use them whenever the field name is more than a single word.
