Dates

SwiftRE has first-class date support. Dates can be compared, added to, subtracted from, and used to compute spans between events.

Date Literals

Explicit dates use the DATE keyword with an ISO 8601 string:

WHEN startDate IS BEFORE DATE "2025-06-15"
THEN SET cutoffDate TO DATE "2025-12-31"

Date References

Built-in references that resolve at execution time:

ReferenceValue
TODAYCurrent date (midnight)
NOWCurrent date and time
YESTERDAYOne day before today
TOMORROWOne day after today

Relative Dates

Express dates relative to the current moment:

SyntaxExampleMeaning
N DAYS AGO7 DAYS AGO7 days before today
N DAYS FROM NOW30 DAYS FROM NOW30 days from today
N MONTHS AGO3 MONTHS AGO3 months before today
N MONTHS FROM NOW6 MONTHS FROM NOW6 months from today
N YEARS AGO1 YEAR AGO1 year before today
N YEARS FROM NOW2 YEARS FROM NOW2 years from today
 
WHEN hireDate IS ON OR AFTER 90 DAYS AGO
THEN INFO "Employee is still in probation period"

Date Arithmetic

Add or subtract durations from any date value or field reference. The result is a date.

TODAY + 30 DAYS
`contractStart` + 1 YEAR
`deadline` - 7 DAYS
DATE "2025-01-01" + 6 MONTHS

In Conditions

WHEN expiryDate IS BEFORE TODAY + 30 DAYS
THEN WARNING "Expiring within 30 days"

WHEN `renewalDate` IS ON OR BEFORE `contractStart` + 1 YEAR
THEN INFO "Contract is within first year"

In SET Actions

THEN SET expiryDate TO TODAY + 90 DAYS
THEN SET renewalDate TO `contractStart` + 1 YEAR
THEN SET reminderDate TO `deadline` - 14 DAYS

Date Difference

Compute the integer span between two dates. The result is a whole number representing the count of days, months, or years.

DAYS BETWEEN startDate AND endDate
MONTHS BETWEEN hireDate AND TODAY
YEARS BETWEEN birthDate AND DATE "2025-01-01"

In Conditions

The date difference expression produces an integer that you can compare like any other number:

WHEN DAYS BETWEEN startDate AND endDate IS GREATER THAN 14
THEN WARNING "Duration exceeds two weeks"

WHEN YEARS BETWEEN birthDate AND TODAY IS LESS THAN 18
THEN ERROR ON age "Must be at least 18 years old"

WHEN MONTHS BETWEEN hireDate AND TODAY IS AT LEAST 12
THEN INFO "Eligible for annual review"

In SET Actions

THEN SET contractDays TO DAYS BETWEEN `contract.start` AND `contract.end`
THEN SET tenure TO YEARS BETWEEN hireDate AND TODAY
THEN SET monthsActive TO MONTHS BETWEEN activationDate AND TODAY

Date Coercion

When comparing a date field against a string value (or vice versa), SwiftRE automatically attempts ISO 8601 parsing. If a fact value like "2025-06-15" is submitted as a string, it will be coerced to a date for comparison against date operators or date literals.

Tip: If your JSON facts contain date strings in ISO 8601 format (yyyy-MM-dd or yyyy-MM-dd'T'HH:mm:ss), they will work seamlessly with date operators. No special formatting is needed on the client side.
