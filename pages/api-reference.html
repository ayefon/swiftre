API Reference

All endpoints use JSON (Content-Type: application/json). Base URL: http://localhost:8080/api/v1

Rulesets

POST /rulesets

Create a new ruleset. The rule text is parsed and validated on creation.

Request body:

{
  "id": "contact-form",
  "ruleText": "RULE \"Email Required\"\nWHEN email IS NOT SET\nTHEN ERROR ON email \"Email is required\""
}

Response (201):

{
  "id": "contact-form",
  "ruleCount": 1,
  "ruleNames": ["Email Required"],
  "hasSchema": false,
  "createdAt": "2025-01-15T10:30:00Z",
  "updatedAt": "2025-01-15T10:30:00Z"
}

curl:

curl -X POST http://localhost:8080/api/v1/rulesets \
  -H "Content-Type: application/json" \
  -d '{"id":"contact-form","ruleText":"RULE \"Test\" WHEN age IS LESS THAN 18 THEN ERROR \"Too young\""}'

Errors: 400 (syntax errors in rule text), 409 (ID already exists)

GET /rulesets

List all rulesets.

Response (200):

[
  {
    "id": "contact-form",
    "ruleCount": 2,
    "ruleNames": ["Age Check", "Email Required"],
    "hasSchema": false,
    "createdAt": "2025-01-15T10:30:00Z",
    "updatedAt": "2025-01-15T10:30:00Z"
  }
]

curl:

curl http://localhost:8080/api/v1/rulesets
GET /rulesets/:id

Get a ruleset with its full rule text.

Response (200): Same as list item, plus "ruleText": "..."

curl:

curl http://localhost:8080/api/v1/rulesets/contact-form

Errors: 404 (not found)

PUT /rulesets/:id

Update a ruleset's rule text.

Request body:

{"ruleText": "RULE \"Updated\" WHEN status IS \"new\" THEN INFO \"Updated rule\""}

curl:

curl -X PUT http://localhost:8080/api/v1/rulesets/contact-form \
  -H "Content-Type: application/json" \
  -d '{"ruleText":"RULE \"Updated\" WHEN status IS \"new\" THEN INFO \"Updated\""}'

Errors: 400 (syntax errors), 404 (not found)

DELETE /rulesets/:id

Delete a ruleset.

Response: 204 (no content)

curl:

curl -X DELETE http://localhost:8080/api/v1/rulesets/contact-form

Errors: 404 (not found)

Execution

POST /rulesets/:id/execute

Execute a ruleset against a set of facts. Nested JSON objects are flattened to dot-notation paths. Supports date arithmetic and date difference expressions.

Request body:

{
  "facts": {
    "firstName": "Jane",
    "age": 16,
    "email": null
  },
  "mode": "singlePass"
}

For iterative mode:

{
  "facts": {"orderTotal": 150},
  "mode": {"iterative": 10},
  "synonyms": {"order total": "orderTotal"}
}

Response (200):

{
  "events": [
    {
      "kind": "error",
      "ruleName": "Age Check",
      "field": "age",
      "message": "Must be 18 or older"
    }
  ],
  "facts": {
    "firstName": "Jane",
    "age": 16,
    "email": null
  },
  "iterations": 1,
  "terminationReason": "complete",
  "summary": {
    "errors": 1,
    "warnings": 0,
    "infos": 0,
    "modifications": 0,
    "total": 1
  }
}

curl:

curl -X POST http://localhost:8080/api/v1/rulesets/contact-form/execute \
  -H "Content-Type: application/json" \
  -d '{"facts":{"age":16,"email":null}}'

Errors: 400 (malformed body), 404 (ruleset not found)

Editor Support

POST /validate

Parse rule text without saving. Always returns 200; check the valid field.

Request body:

{
  "ruleText": "RULE \"Test\" WHEN age IS LESS THAN 18 THEN ERROR \"Too young\"",
  "rulesetId": "contact-form"
}

rulesetId is optional — if provided, schema warnings are included.

Response (valid):

{
  "valid": true,
  "ruleCount": 1,
  "ruleNames": ["Test"],
  "referencedFields": ["age"]
}

Response (invalid):

{
  "valid": false,
  "error": "Expected condition after WHEN, got end of input"
}

curl:

curl -X POST http://localhost:8080/api/v1/validate \
  -H "Content-Type: application/json" \
  -d '{"ruleText":"RULE \"Test\" WHEN age IS LESS THAN 18 THEN ERROR \"Too young\""}'
GET /schema/operators

Returns all comparison operators grouped by data type, plus date value references, date arithmetic forms, and date difference forms.

curl:

curl http://localhost:8080/api/v1/schema/operators
GET /schema/actions

Returns all action types with syntax and descriptions.

curl:

curl http://localhost:8080/api/v1/schema/actions
GET /rulesets/:id/synonyms

Get synonym mappings for a ruleset.

Response (200):

{"synonyms": {"customer age": "demographics.age"}}

curl:

curl http://localhost:8080/api/v1/rulesets/contact-form/synonyms
PUT /rulesets/:id/synonyms

Update synonym mappings for a ruleset.

Request body:

{
  "synonyms": {
    "customer age": "demographics.age",
    "zip code": "address.zip"
  }
}

curl:

curl -X PUT http://localhost:8080/api/v1/rulesets/contact-form/synonyms \
  -H "Content-Type: application/json" \
  -d '{"synonyms":{"customer age":"demographics.age"}}'
GET /rulesets/:id/fields

Returns all fact paths known for a ruleset, derived from rule conditions/actions and prior executions.

Response (200):

{
  "fields": [
    {"path": "age", "synonym": "customer age", "lastSeenType": "integer"},
    {"path": "email", "synonym": null, "lastSeenType": "string"}
  ]
}

curl:

curl http://localhost:8080/api/v1/rulesets/contact-form/fields
GET /rulesets/:id/schema

Get the data model schema for a ruleset.

curl:

curl http://localhost:8080/api/v1/rulesets/contact-form/schema
PUT /rulesets/:id/schema

Update the data model schema. See Data Model: Schema Validation.

curl:

curl -X PUT http://localhost:8080/api/v1/rulesets/contact-form/schema \
  -H "Content-Type: application/json" \
  -d '{"fields":{"age":{"type":"integer","required":true}}}'

Error Responses

All errors follow the same format:

{
  "error": true,
  "reason": "Ruleset 'missing-id' not found"
}
StatusMeaning
400Bad request (malformed JSON, syntax errors in rule text)
404Ruleset not found
409Conflict (ruleset ID already exists)
501Not implemented (reserved endpoints)
