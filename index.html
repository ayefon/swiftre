SwiftRE

A business rules engine you can read. Write validation logic in plain English, execute it via a REST API, and get structured events for your application.

📝

English-like Rules

Write rules that read like sentences. No code compilation, no DSL to learn — just WHEN / THEN logic that business stakeholders can review.

⚡

REST API

Create rulesets, execute them against JSON facts, and get results — all through a simple HTTP API. Integrate from any language or platform.

🔔

Event-driven Output

Rules produce typed events (errors, warnings, info, signals) instead of side effects. Your application decides how to present and handle each event.

Hello World

A simple validation ruleset that checks a contact form submission:

RULE "Email Required"
WHEN email IS NOT SET
THEN ERROR ON email "An email address is required"

RULE "Age Check" PRIORITY 10
WHEN age IS SET AND age IS LESS THAN 18
THEN ERROR ON age "You must be at least 18 years old"

RULE "Welcome"
WHEN firstName IS SET
THEN INFO "Welcome, {firstName}!"

Execute this ruleset with facts like {"age": 16, "email": null, "firstName": "Jane"} and you get structured events:

{
  "events": [
    {"kind": "error", "ruleName": "Email Required",
     "field": "email", "message": "An email address is required"},
    {"kind": "error", "ruleName": "Age Check",
     "field": "age", "message": "You must be at least 18 years old"},
    {"kind": "info",  "ruleName": "Welcome",
     "message": "Welcome, Jane!"}
  ],
  "summary": {"errors": 2, "warnings": 0, "infos": 1, "total": 3}
}
